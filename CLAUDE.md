# CLAUDE.md

このファイルはプロジェクト固有の運用ルールを記録する。Claude Codeは本ファイルの内容に常に従うこと。

## セッションログの自動MD化・Skill化ルール

### 目的

チャットでのやり取りが一区切りついた（1つの作業・調査・実装が完了した）時点で、
確認を挟まず自動的にセッションログを記録し、必要に応じてSkillへ切り出す。

### ルール

1. **セッションログのMD化**
   - 作業が一区切りついたと判断したら、その都度
     `docs/sessions/YYYY-MM-DD_<内容を表す短い英数字スラッグ>.md`
     というファイルを新規作成する（既存ファイルへの追記ではなく1トピック1ファイル）
   - 内容は以下の構成にする:
     - 背景・依頼内容
     - 実施したこと（変更したファイル、実行したコマンド、判断の理由）
     - 結果・現状
     - 未解決の課題・次のアクション

2. **Skill化の判断**
   - 上記の作業が「今後も繰り返し使えるパターン」
     （例: 特定のレビュー観点、特定の資料作成手順、特定のデバッグ・検証手順）
     だと判断した場合のみ、このプロジェクト内
     `.claude/skills/<skill名>/SKILL.md` として切り出す
   - SKILL.mdの形式:
     ```
     ---
     name: <skill名>
     description: "<いつ使うべきかを1〜3文で。トリガーになる状況を具体的に>"
     ---
     # <タイトル>
     <手順・観点・チェックリストなど、再利用可能な形で>
     ```
   - 単発・一度きりの修正はSkill化しない。判断に迷う場合はSkill化せず、
     MDファイルの末尾に「Skill化候補: <理由>」と一行だけ書き添えるにとどめる
   - Skillを新規作成/更新した場合は、その旨をチャットの返答内で一言報告する

3. **確認は挟まない**
   - 1と2は確認を取らず自動で実行してよい
   - ただし、既存コードの変更・ファイル削除など元に戻せない操作は
     従来通り確認を挟む（このルールでは変更しない）
