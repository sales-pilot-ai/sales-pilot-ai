# Sales Pilot AI 開発進捗

最終更新日: 2026-07-03

---

## プロジェクト概要

**Sales Pilot AI** は、営業担当者が毎日使うことを前提にした、新規開拓営業の半自動化ツールです。

- Google Maps / Web検索で企業情報を収集する
- 企業サイトを解析して連絡先など不足情報を補完する
- Google Sheets を「正」とする営業リストとして一元管理する
- Gmail からパーソナライズしたメールを一括送信する
- 送信後の返信検知・商談・成約/失注までの営業ライフサイクルを CLI だけで完結させる

一貫した設計思想は **「Google Sheets を管理するシステム」ではなく「担当者が普段使う Sheets と共存するツール」** であること。人が手入力した値（送信可否・メモ・返信有無など）は保護され、AI が自動上書きすることはありません（詳細: [ADR-001](adr/ADR-001-google-sheets-as-source-of-truth.md)）。

### 現在どこまで完成しているか

2026-06 の基盤構築（企業検索・Sheets連携・Gmail送信・パーソナライズ）に続き、2026-07 は「送信して終わり」ではなく **送信後のライフサイクル（返信 → 商談 → 成約/失注）まで CLI だけで回せること** を最優先に開発を進めてきました。

現時点（Issue #032 完了時点）で、営業担当が日々の業務（企業探し・送信・返信管理・進捗更新・状況把握・フォロー漏れ防止・テンプレート出し分け）をすべて CLI から行える状態になっています。

## 現在できること

営業担当者が実際に使える機能です。

- **企業検索**（`find`）— 業種・地域を指定して企業情報を収集し、Webサイトから連絡先等を自動補完
- **レビュー付き追加**（`find --review`）— 検索結果を1社ずつ確認してから営業リストへ保存
- **メール送信**（`send`）— パーソナライズした営業メールを一括送信（`--dry-run`/`--force`/`--template`対応）
- **送信プレビュー**（`send --preview`）— 送信前にテンプレート・宛先・件名を一覧確認し、1社ずつ本文全文の確認や「今回だけ送信対象から外す」判断が可能
- **返信検知**（`check-replies`）— Gmailの返信を自動検知して営業リストへ反映
- **ステータス更新**（`update`）— 商談日設定・成約・失注・メモ追記をシートを開かずに実行
- **営業レポート**（`report`）— 送信率・返信率・商談/成約/失注件数などの集計を表示
- **フォローアップ管理**（`follow-up`）— 今日やるべき商談・返信フォローを優先順位順に表示
- **テンプレート管理**（`template`）— 複数のメールテンプレートを作成・編集・複製・削除
- **営業ダッシュボード** — Sheets上に自動集計タブを生成し、ブラウザからも状況把握可能
- **送信履歴の記録** — 送信結果（成功/失敗/スキップ）をSheetsに記録し追跡可能

## システム構成

```
                 営業担当者
                     │
                     ▼
            CLI（sales-pilot / commander）
                     │
   ┌─────────────┼─────────────────┬───────────────┐
   ▼                 ▼                 ▼                 ▼
Google Maps /    Google Sheets     Gmail API        テンプレート
Search API       （Source of      （送信 /          （templates/emails/
（企業情報収集）   Truth）           返信検知）         + templates.json）
                 営業リスト・
                 送信履歴・
                 返信履歴・
                 ダッシュボード
```

- **CLI**: `find`/`send`/`update`/`report`/`follow-up`/`template` などのコマンドを提供。ビジネスロジックはService層（`SheetsService`/`templates/manager.js`等）に集約し、CLIは表示に専念する設計を徹底
- **Google Sheets**: 営業リスト（企業データ）・送信履歴・返信履歴・ダッシュボードタブを保持する唯一の正データ。ヘッダー駆動で列順に依存しない
- **Gmail API**: メール送信と返信検知（`check-replies`）に使用
- **テンプレート**: `templates/emails/` 配下のファイル + `templates.json`（メタデータ）で複数テンプレートを管理

## 現在の完成度

**社内利用可能**

企業探しから送信、そして送信後のライフサイクル（返信検知 → 商談設定 → 成約/失注記録）と日々の状況把握（レポート・フォローアップ）まで、営業担当が実務で必要な一連の操作をシートを開かずにCLIだけで完結できる段階に達しています。MVP（最小限の送信フロー）は完了済みで、現在は日次運用に必要な機能（優先順位付け・複数テンプレート・確認体験の改善）を継続的に強化しているフェーズです。

外部の複数チーム・複数ユーザーでの本格運用や、AIによる自動パーソナライゼーション・通知連携（#033〜#036）はまだ今後の課題であり、その意味では **β版の一歩手前** の位置づけです。
