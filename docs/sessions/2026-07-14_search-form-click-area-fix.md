# 企業検索画面: クリック可能領域が見た目より狭い問題への対応

## 背景・依頼内容

実機で、企業検索画面（Search.html）の業種・エリア・取得件数の入力欄、「検索実行」ボタンの
すべてで、クリックできる領域が見た目のサイズより狭いという報告を受けた。原因特定と修正、
および同じレイアウト構造の他画面の確認を依頼された。

## 調査

Playwright（Chromiumヘッドレス）で、実際のStyle.htmlのCSS＋Search.htmlの構造を再現した
静的ページを作成し、業種入力欄・検索実行ボタンの四隅と中心付近をクリックして、実際に
フォーカス／クリックイベントが登録されるかを実測した（デスクトップ幅1280px・モバイル幅375px
の両方で検証）。

**結果: 現状（変更前）のCSS・HTMLでは、入力欄・ボタンいずれも、見た目のボックスの
四隅まで含めてクリック・フォーカスが正しく機能しており、視覚的な境界と実際のヒット領域の
ズレを再現できなかった。** `writing-mode`等の縦書き指定、transform、負のmargin、
transparentなオーバーレイの重なりのいずれも見当たらず、`box-sizing: border-box`も
グローバルに適用済みだった。

このため、実機での症状の正確な原因はコード上・ヘッドレスブラウザ上では特定できていない。
GASのWebアプリはiframeサンドボックス内で描画される仕組みのため、認証済みの実環境でしか
再現できない、iframe特有の座標変換・ビューポート差異等が残る可能性がある。

## 実施したこと（原因を確定できなかったため、確実に安全で意味のある改善のみ実施）

- `Search.html`: 業種・エリア・取得件数の入力欄を、labelでinputを暗黙的にラップする方式
  （`<label>業種 <input /></label>`）から、`for`属性による明示的な関連付け
  （`<label for="search-industry">業種</label><input id="search-industry" />`）に変更。
  レイアウトは新設した`.search__field`ラッパーでlabel+inputのひとまとまりを維持し、
  見た目は変えていない
- 「同じレイアウト構造の他画面」の確認: `Settings.html`・`Templates.html`・
  `CompanyList.html`（下書きモーダル・ステータス変更モーダル）に、`<label>テキスト</label>`
  の直後に別要素として`<input>`/`<select>`/`<textarea>`が続くだけで、暗黙のネストも
  `for`属性による関連付けも一切無い箇所が多数見つかった（**これはSearch.htmlより
  明確に悪い状態で、ラベルテキストをクリックしても何も起きない**）。該当する全ラベルに
  `for`属性を追加した

## 実施した検証

1. `npm run lint` → エラー0件
2. `npm test` → 40ファイル/806テスト 全てPASS
3. Playwrightによる実測（新旧両方のマークアップで実施）
   - 修正後の`.search__field`構造でも、入力欄の四隅クリックで正しくフォーカスされることを確認
   - `for`属性によるラベルテキストクリックでも、対応する入力欄に正しくフォーカスされることを確認
   - チェックボックスラベル（`for`追加後も引き続きinputをラップする形）の、テキスト部分の
     右端をクリックしてもチェックが切り替わることを確認
   - 検索実行ボタンの四隅クリックがすべて正しく登録されることを確認（`event.preventDefault()`を
     伴う実際のフォーム送信ハンドラーを含めた状態で再検証）
