# 左サイドバーの固定表示化

## 背景・依頼内容

左サイドバー（Dashboard・営業管理・テンプレート・フォローアップ・レポート・設定、
下部のアカウント表示）が、メインコンテンツのスクロールに連動して画面外に隠れてしまう
という報告を受けた。サイドバーを固定し、メインコンテンツ側だけがスクロールする
2カラムレイアウトにしたい、という依頼。全画面共通のレイアウトに適用し、既存の
モバイル幅レスポンシブ対応（サイドバー折りたたみ）は壊さないことが条件。

## 調査

`Views/Index.html`が全画面共通のレイアウトテンプレートで、`<main class="app-main">`内に
`<?!= include(pageTemplate); ?>`で各画面（Dashboard/CompanyList/Templates/Report/Settings等）
を差し込む構造になっている。共通CSSは`Views/Style.html`の`.app-shell`（flexコンテナ）・
`.app-sidebar`（幅240pxのflexアイテム）・`.app-main`（flex:1）。

`.app-shell`にも`.app-sidebar`にも高さの上限・overflow指定が一切無く、
`.app-shell`は`align-items: stretch`（デフォルト）のflexコンテナのため、サイドバーの
高さは常に`.app-main`側のコンテンツの高さに引き伸ばされる形になっていた。コンテンツが
ビューポートより長くなると、ページ全体（body）がスクロールし、サイドバーもその一部として
一緒に流れていってしまう、というのが原因だった。

## 実施したこと

`Views/Style.html`の`.app-sidebar`（デスクトップ向けルール）に以下を追加：

```css
position: sticky;
top: 0;
height: 100vh;
overflow-y: auto;
```

`position: sticky`はflexレイアウト上の通常フローに残ったまま、スクロール時に
ビューポートの上端に張り付く。`height: 100vh` + `overflow-y: auto`により、サイドバー
自身の内容がビューポートより長くなった場合のみサイドバー内で独立スクロールし、
ページ全体の二重スクロールにはならない。

`@media (max-width: 768px)`内の`.app-sidebar`ルール（既存のモバイル横並び折りたたみ
レイアウト）には、デスクトップ向けの固定表示を打ち消す`position: static`・
`height: auto`・`overflow-y: visible`を追加し、モバイル幅では従来どおりの挙動に
戻るようにした。

`Views/Index.html`（共通レイアウトテンプレート）自体、および各画面（Dashboard.html等）の
HTML構造・ロジックは一切変更していない。CSS（`Style.html`）のみの変更。

## 実施した検証

1. `npm run lint` → エラー0件
2. `npm test` → 40ファイル/806テスト 全てPASS
3. Style.htmlの中括弧の対応数チェック → 一致（186/186、変更前後で不変）
4. Playwrightによる実機相当の検証（実際のStyle.htmlのCSSをそのまま抽出し、
   Index.htmlのサイドバー構造を再現した静的ページで検証）
   - デスクトップ幅（1280px）: メインコンテンツを2000pxスクロールしても、サイドバーの
     上端座標が0のまま（画面内に固定表示）で、ログアウトボタン等も引き続き見えることを確認
   - サイドバーの項目を大量に追加してビューポートより長くなる状況を再現し、サイドバー自体は
     ビューポート内の高さに収まり、`overflow-y: auto`で独立スクロール可能になることを確認
   - モバイル幅（375px）: サイドバーが`position: static`・横並び（`flex-direction: row`）に
     戻り、ナビは初期状態で折りたたまれている（`display: none`）ことを確認し、既存の
     レスポンシブ挙動が変更されていないことを確認
