# 営業リストタブに企業がほぼ表示されない不具合の修正

## 背景・依頼内容

実機での確認により、営業リストシートには14社分の行（ヘッダー直後の企業ID空の
「テスト株式会社」＋C000001〜C000013）が存在するにもかかわらず、アプリの営業リスト
タブにはC000007（送信状況「送信済」）の1社しか表示されないという報告を受けた。
表示される・されないの差分は送信状況（送信済 or 未送信）のみで、メールアドレスの
有無とは無関係と切り分け済みだった。

（本件は直前に報告された「version 93の一括保存がシートに書き込まれていない」の
更新版として届いたもの。調査の結果、保存自体は正しく行われており、問題は表示側の
絞り込み条件にあったことが本人により特定されていた。）

## 調査結果（原因）

`Views/CompanyList.html`の`applyTabBaseFilter`関数（Sprint8④-1で導入）が

```js
function applyTabBaseFilter(companies) {
  return companies.filter(function (c) {
    return c.sendStatus !== '未送信';
  });
}
```

という条件で、**営業リストタブは「送信済（＝初回メール送信済み）」の企業のみを表示し、
未送信の企業は一括送信タブの対象とする**という設計になっていた。これ自体は
Sprint8時点での意図的な仕様だったが、本セッション内で別途実施した「送信状況（送信状況列）
が空欄のまま保存されるバグの修正＋既存データへのbackfill」により、以前は空欄だった
未送信企業の送信状況が正しく「未送信」という文字列で保存されるようになった結果、
このタブ基底フィルターに引っかかって表示から除外されるようになっていた
（＝コード自体は仕様通り動作しているが、直前のバグ修正の副作用で「ほぼ表示されない」
という体感になっていた）。

企業ID空の「テスト株式会社」行によるパース中断（仮説2）は、`rowToCompany_`が
値の欠落を単に空文字で埋めるだけで例外を投げない実装のため、当てはまらないことを
コード確認で確認した。

## 実施したこと

- `Views/CompanyList.html`: `applyTabBaseFilter`を、送信状況によるフィルタを撤廃し
  シート上の全企業をそのまま表示するよう変更（一括送信タブ側の絞り込み条件
  `loadSendTab`は変更していない。別関数・別データフローのため本修正の影響を受けない）
- `Services/SheetsService.js`: `listCompaniesFromSheet_`で、企業IDが空の行
  （テストデータ等の不正行）を一覧結果から除外するようにした。除外はエラーを
  投げず、`Logger.log`でスキップした行番号を記録するのみで、他の行の処理には
  一切影響しない
- メールアドレスが空の企業の扱い（一覧には表示し空欄表示、メール送信系アクションは
  対象外としエラー表示）は、`GmailService.js`/`SendMailService.js`が既に
  `company.email`の有無をチェックして明確な日本語エラーを投げる実装になっており、
  クライアント側の`.withFailureHandler`がそのエラーメッセージをトースト表示する
  作りに既になっていたため、追加の変更は不要だった（前回依頼分は対応済み）

     受けず、従来どおり未送信12社のままであることを確認
