# 返信確認タブの表示条件確認＋送信・返信ステータスの色分け

## 背景・依頼内容

version 94（営業リストタブ全企業表示化）を前提に、営業管理3タブの役割を再確認したい
という依頼。タスク1は返信確認タブの表示条件がコードから見て「返信が来た企業のみ表示」
になっているかを確認し、なっていなければ修正すること。タスク2は営業リストタブの一覧で、
送信状況・返信有無の値に応じて文字色を変える依頼（レイアウト・背景・シート上のデータは
変更しない）。

## タスク1: 返信確認タブの表示条件確認

`Views/CompanyList.html`の返信確認タブは「新着返信」「継続返信」の2サブタブで構成されており、
現在の条件は以下のとおりだった（コード確認のみ、変更前）:

- 新着返信（`loadNewReplyTab`）: `c.hasReply === '返信あり' && c.sendStatus !== '対応中' && !c.dealResult`
- 継続返信（`loadOngoingReplyTab`）: `c.hasReply === '返信あり' && c.sendStatus === '対応中'`

両サブタブとも`hasReply === '返信あり'`を必須条件としており、2つを合わせた「返信確認タブ」
全体としては、依頼の仕様どおり**返信が来た企業（返信有無＝返信あり）のみを表示する**
仕様に既に合致していた。**このためコード変更は行っていない。**
（営業リストタブ・一括送信タブの表示条件も、今回の確認・修正の対象外として変更していない）

## タスク2: 送信・返信ステータスの色分け

### 実施したこと

- `Views/CompanyList.html`: 営業リストタブの行テンプレートに、送信状況・返信有無セルの
  表示専用関数`renderSendStatusCell(c)`・`renderHasReplyCell(c)`を追加し、直書きの
  `escapeHtml(c.sendStatus)`・`escapeHtml(c.hasReply)`を置き換えた
  - `c.sendStatus === '未送信'` → `<span class="status-text--danger">未送信</span>`
  - `c.sendStatus === '送信済'` → `<span class="status-text--info">送信済み</span>`
    （表示ラベルのみ「送信済み」にしている。シート上の値・`c.sendStatus`自体・フィルター
    判定に使う文字列は一切変更していない）
  - 上記以外の値（対応中・保留・送信失敗・配信停止・空欄等）は`escapeHtml(c.sendStatus)`
    のまま、従来どおりの表示
  - `c.hasReply === '未返信'` → 赤span、`c.hasReply === '返信あり'` → 青spanで同様に処理。
    それ以外・空欄は従来どおり
- `Views/Style.html`: `.status-text--danger`（`var(--color-error)`）・
  `.status-text--info`（`var(--color-accent)`）を追加。既存のエラー色・アクセント色
  （プライマリボタン等で使用中の青）をそのまま流用し、新しい色定義は追加していない

### 適用範囲の確認

送信状況・返信有無を生の値として表示しているのは営業リストタブの一覧テーブルのみだった。
一括送信タブ（`renderSendTable`）の表示列は会社名・担当者名・メールアドレス・業種のみ、
返信確認タブ（新着返信・継続返信）はカード形式で会社名・担当者名・本文プレビュー等を
表示するのみで、いずれも送信状況・返信有無を列として表示していないため、**この2箇所への
適用は不要（対象なし）**と確認した。
