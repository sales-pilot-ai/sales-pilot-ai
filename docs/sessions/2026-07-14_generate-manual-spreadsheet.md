# 取扱説明書をGoogleスプレッドシートとして生成するセットアップ関数の追加

## 背景・依頼内容

`docs/manual-source.md` の内容を、社内配布用のGoogleスプレッドシート（既存の営業データの
スプレッドシートとは別の新規スプレッドシート）として出力したいという依頼。Apps Scriptエディタから
手動実行する `generateManualSpreadsheet()` を新規ファイルとして追加し、画面一覧・操作フロー・
ボタン説明・権限の違い・注意点・よくあるエラーの6タブ＋表紙タブを作成、見た目（ヘッダー太字＋
背景色＋固定表示、罫線、列幅自動調整、ステータスキーワードの色分け）を整え、共有設定
（ドメイン内リンク閲覧可）まで行うことが求められた。アプリ本体（Router/Service/View）には
影響を与えないこと、既存デプロイも不要という制約。

## 実施したこと

- `gas/src/Scripts/GenerateManual.js` を新規作成。Apps Script実行環境は
  `docs/manual-source.md` を実行時に読み込めない（`clasp push`は`gas/src/**`のみを対象とし、
  リポジトリ内の他ディレクトリはアップロードされない）ため、原文の内容をこのファイル内に
  JS配列・オブジェクトとして書き写す形にした（要約・省略はしていない。データの行数は
  `docs/manual-source.md` の各表・箇条書きの行数と完全に一致させている）
- 主な構成:
  - `buildManualRichText_(rawText)`: `docs/manual-source.md`の`**太字**`記法をSheetsの
    リッチテキストの太字表示に変換しつつ、「未送信」「未返信」を赤系（`#d93025`）、
    「送信済み」「返信あり」を青系（`#1a73e8`）にする（Views/Style.htmlの
    `--color-error`/`--color-accent`と同じ値を使用し、既存アプリの色分けと合わせた）
  - `writeManualTable_`: ヘッダー太字＋背景色（`#e8f0fe`）＋固定表示、罫線、列幅自動調整を
    適用しつつ表を書き込む共通処理（画面一覧・操作フロー・ボタン説明・権限の違い・
    よくあるエラーの5タブで使用）
  - `writeManualList_`/`appendManualNote_`: �655線を伴わない見出し＋本文リスト（注意点タブ、
    各表下部の補足）
  - 操作フロー・ボタン説明タブは、`docs/manual-source.md`の各画面ごとの番号付き手順・表を
    「画面」列を追加して1枚の表にまとめる形にした（すべての手順・項目を保持したまま、
    タブとして扱いやすい形に整理。情報の省略はしていない）
  - 「レポート」画面はmanual-source.md上は表ではなく一文の説明のみだったため、ボタン説明タブ
    では同じ情報を1行の表形式（項目=(なし)、種別=-、機能=原文の説明文）として表現した
    （情報を保ったまま形式を統一するための対応であり、内容の省略ではない）
  - `generateManualSpreadsheet()`: `SpreadsheetApp.create()`で新規スプレッドシートを作成し、
    表紙（左端）→画面一覧→操作フロー→ボタン説明→権限の違い→注意点→よくあるエラーの順で
    タブを構成。表紙タブにはタイトル・対象バージョン（version 96）・作成日・各タブへの
    `HYPERLINK`式による目次リンクを設置。最後に`DriveApp.getFileById(...).setSharing(
    DriveApp.Access.DOMAIN_WITH_LINK, DriveApp.Permission.VIEW)`で共有設定し、
    `Logger.log()`でURLを出力する
- 既存の営業データスプレッドシート・Router.js・Services・Viewsへの参照や書き込みは一切ない
  （`SpreadsheetApp.create()`で完全に独立した新規ファイルを作るのみ）

     総数と一致することをgrepでも別途確認済み）が省略なく書き込まれること
   - ボタン説明タブ: 全8画面・59行（レポート画面の1行分の形式変換による+1を除き、

   - 共有設定が`DOMAIN_WITH_LINK`/`VIEW`で呼ばれ、`Logger.log`にURLが出力されること

## 結果・現状

`gas/src/Scripts/GenerateManual.js`を新規作成し、コミット・push済み。デプロイは行っていない
（依頼の制約どおり、既存Webアプリの動作・デプロイには一切影響しない）。

## 未解決の課題・次のアクション

- 実際にApps Scriptエディタから`generateManualSpreadsheet()`を実行し、生成されるスプレッドシートの
  見た目・共有設定・目次リンクの動作を確認する必要がある（Node vmシミュレーションはAPIの
  呼び出し内容・データの正しさを検証したものであり、実際のGoogle Sheets上の見た目までは
  確認できていない）
- `docs/manual-source.md`の内容が今後更新された場合、`GenerateManual.js`内のデータ定義も
  あわせて更新する必要がある（自動連携の仕組みは無い）
