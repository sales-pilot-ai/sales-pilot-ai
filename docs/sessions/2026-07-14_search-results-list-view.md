# 企業検索結果の一覧表示化・複数選択保存への変更

## 背景・依頼内容

企業検索（Search.html）の検索結果確認が「候補 1/20」のように1社ずつのカード表示で、
20社確認するのに20回操作が必要だった。検索結果を縦一列の一覧表示にし、チェックボックスで
複数選択して一括保存できるようにする依頼。あわせて、一覧化によりWebサイト解析が
まとめて走るとGASの6分実行制限に抵触する可能性があるため、安全な解析方式を検討・採用する
よう指示された。

## 採用した解析方式: 「検索時は解析しない。保存直前に選択企業ごとに個別解析する」

`Services/MapsService.js`の`searchCompaniesViaMaps_`を確認したところ、既存の実装では
検索実行の**単一のRouter呼び出しの中で**、ヒットした全候補に対して順番に
`analyzeWebsite_`（外部サイトへのHTTPフェッチ＋HTML解析）を実行していた。一覧表示に
変えても変えなくても、この「検索1回＝全候補分の解析が単一実行内で走る」という設計は
変わらず存在しており、取得件数の上限（50件）まで多くの解析対象サイトが遅い場合、
GASの6分実行制限に抵触するリスクは元々あった。

このリスクを避けるため、以下の方式を採用した：

- 検索実行時（`searchCompanies`）は、**常に**`skipAnalyzer: true`を指定して呼び出すよう
  Search.html側を変更した。`searchCompaniesViaMaps_`自体のコード・仕様は一切変更していない
  （既存の`skipAnalyzer`引数をそのまま使っているだけ＝「呼び出し方の変更」）
- 保存は選択企業ごとに1件ずつ、個別の`google.script.run`呼び出しで実行する
  （既存の一括送信タブ`runSendTabBatch`と同じ、クライアント側で順に呼ぶパターンを踏襲）。
  ユーザーが「Webサイト解析をスキップする」を指定していない場合のみ、保存の直前に
  新設した`analyzeCandidateWebsite(websiteUrl)`（既存の`analyzeWebsite_`をそのまま
  呼ぶだけのRouter薄いラッパー）でその1件だけを解析し、結果を`candidate`へ反映してから
  既存の`saveCompanyCandidate`（変更なし）を呼ぶ
- これにより、1回のサーバー実行が処理する解析対象は常に最大1件になり、選択件数が
  何件であってもGASの6分実行制限に抵触しない設計になっている（各`google.script.run`
  呼び出しはそれぞれ独立して新しい実行時間の枠を持つため）
- 「Webサイト解析をスキップする」チェックボックスの意味は変えていない
  （チェック時は保存時も解析を一切行わない。チェックしない場合は保存対象企業のみ
  個別に解析される。従来「検索時に全件解析」だったタイミングが「保存時に選択企業のみ
  解析」に変わっただけ）

## 実施したこと

- `Views/Search.html`: 1社ずつのカードレビュー（候補1/20表示、スキップ／保存／
  残りをすべて保存／終了するボタン）を全面的に一覧表示＋複数選択保存に置き換えた
  - 検索結果を縦に並べたカード一覧として描画（既存のカードのデザイン・表示項目を流用）
  - 各カードにチェックボックスを追加。デフォルト全社チェックON
  - 「全選択」「全解除」ボタンを追加
  - 「選択した企業を保存」ボタンで、選択された企業のみを1件ずつ処理
    （解析→保存、または解析スキップなら保存のみ）
  - 保存中は「保存中… n/m件」の進捗バー表示（既存の`.company-list__bulk-progress`/
    `.progress-bar`をそのまま流用）
  - 保存が完了したカードはチェックボックスをdisabledにし、結果ラベル
    （追加しました／既存企業を更新しました／変更なし／保存に失敗しました）を表示。
    未選択のまま残った企業はチェックボックスが有効なままなので、保存済み・未保存が
    見た目で判別できる
  - 完了後のサマリーは既存同様、追加／重複更新／変更なし／失敗／未保存の件数を表示
  - 不要になった「スキップ」「残りをすべて保存」「終了する」ボタンとそのハンドラーを削除
- `Router.js`: `analyzeCandidateWebsite(websiteUrl)`を新規追加
  （既存の`analyzeWebsite_`をそのまま呼ぶだけ）
- `Views/Style.html`: `.search__list-toolbar`・`.candidate-card__header`・
  `.candidate-card__result`（+状態別の色分け）等を追加。不要になった
  `.candidate-card__actions`・`.search__bulk-actions`は削除
- `eslint.config.js`: `analyzeWebsite_`をgasProjectGlobalsに追加
  （Router.jsから別ファイル定義の関数を呼ぶため）

`saveAllCompanyCandidates`（Router.js、既存の一括保存関数）は今回のUI変更後は
どの画面からも呼ばれなくなったが、Sprint8の`sendSalesMailBatch_`と同様の判断で、
Router関数自体の削除は今回のタスク範囲を超える変更になるため削除せず残した。
