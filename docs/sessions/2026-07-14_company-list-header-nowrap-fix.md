# 営業リストテーブルヘッダーの縦折り返し修正

## 背景・依頼内容

version 89デプロイ後、営業リストタブの実機確認で、テーブルヘッダーの文字が列によって
縦（1文字ずつ折り返し）と横で混在していることが判明。横書き・1行表示に統一し、
必要に応じてラベルを短縮、それでも収まらない場合は横スクロールを許可してほしいという依頼。

（補足: この依頼の直前、同内容をWorkflowツール（investigate→implement→verify の3エージェント
構成）で実行しようとしたが、3エージェントとも「セッション利用上限」により失敗し、
何も変更されないまま終了した。そのため、本タスクは最初からやり直し、通常の直接作業として
実施した。)

## 調査結果（原因）

- `Style.html`のどこにも`writing-mode`/`text-orientation`は存在しない（縦書き指定が原因ではなかった）
- `.company-table`は`table-layout`未指定（ブラウザ標準のauto）、`.company-table th`にも
  `white-space`指定が無かった。日本語ラベルは単語区切り（スペース）が無いため、列幅が
  自動計算で狭くなった列（データ内容が短い列）では見出しが文字単位で折り返されていた
- `white-space: nowrap`は`@media (max-width: 768px)`内の`.company-table`ルールにしか
  存在せず、デスクトップ幅では一切適用されていなかった
- `.company-list`/`.sales-tab-panel`には専用のCSSが無く、テーブルの直接の親要素には
  横スクロールの仕組みが存在しなかった

## 実施したこと

- `Views/CompanyList.html`：`#company-table`のtheadのみ変更
  - `<table>`を新規`<div class="company-table__scroll">`でラップ（id/class/子要素は変更なし。
    既存のJS（`getElementById('company-table')`等）はそのまま動作する）
  - ラベル短縮＋`title`属性付与（4件）：
    「送信状況」→「送信」、「返信有無」→「返信」、「現在ステータス」→「ステータス」、
    「メールアドレス」→「メール」
  - 会社名／担当者／業種／エリア／商談日／成約／最終更新／詳細／AI要約／AI操作は、
    nowrap適用後は列幅不足があっても折り返されなくなる（横スクロールで吸収されるため）ため、
    依頼にない追加の短縮は行わなかった
- `Views/Style.html`
  - `.company-table__scroll { overflow-x: auto; }` を新規追加（デスクトップ幅でも常時有効）
  - `.company-table th`に`white-space: nowrap;`を追加（全幅で常時ヘッダーの折り返しを禁止）
  - `.company-table th:nth-child(2)`（会社名列）に`min-width: 140px;`を追加
  - 既存のモバイル専用`.company-table { display:block; overflow-x:auto; white-space:nowrap; }`
    ルールはそのまま変更していない（回帰リスクを避けるため）
