# 取扱説明書に「アプリへのアクセス方法・ログイン手順」を追加

## 背景・依頼内容

`docs/manual-source.md`および生成済みスプレッドシートを確認したところ、「そもそもアプリを
どうやって開いてログインするか」の説明が抜けていることが判明したため、各画面の操作フローの
先頭に「2-0. アプリへのアクセスとログイン」を追加する依頼。権限シートの存在やApps Scriptの
実装詳細には触れず、利用者向け情報のみとすることが条件。

## 実施したこと

- `docs/manual-source.md`: 「## 2. 各画面の操作フロー」の直後、「### 2-1. Dashboard」の
  前に「### 2-0. アプリへのアクセスとログイン」を新設し、以下の6ステップを追加した:
  1. 管理者から共有されたURL（このアプリのWebアプリURL）にブラウザでアクセスする
  2. Googleアカウントのログイン画面が表示された場合は、会社で使用しているGoogleアカウントを
     選択してログインする
  3. 初回アクセス時、まだ利用者として登録されていない場合は「権限が設定されていません」という
     画面が表示されることがある。その場合は管理者（Admin権限を持つ利用者）に連絡し、
     利用者登録を依頼する（「6. よくあるエラーと対処」の該当項目も参照、と明記し整合させた）
  4. ログインに成功すると、左サイドバーとDashboard画面が表示される
  5. 毎回ログインし直す必要はなく、通常はブラウザにログイン状態が保持されたまま次回からも
     そのまま開ける
  6. URLはブラウザにブックマークしておくと、次回から素早く開ける
- `gas/src/Scripts/GenerateManual.js`: `MANUAL_FLOW_SECTIONS`配列の先頭に、上記と同じ内容の
  `{ screen: 'アプリへのアクセスとログイン', steps: [...] }`を追加した。既存の
  `Dashboard`以降のセクションはそのまま（各セクションの手順番号はセクションごとに1から
  振り直される実装のため、既存セクションの番号をずらす必要はなかった）
- 他の章・タブ（画面一覧、ボタン説明、権限の違い、注意点、よくあるエラー）の内容は
  変更していない
