# Architecture Decision Records (ADR)

重要な設計・技術的判断を記録するためのドキュメント群。

---

## ADR とは

「なぜその判断をしたか」を残すためのもの。

- 実装の詳細はコードに書く
- 判断の経緯・背景・却下した選択肢はここに書く

コードを読んでも分からない「理由」だけを記録する。

---

## ファイル命名規則

```
ADR-NNN-kebab-case-title.md
```

例: `ADR-001-google-sheets-as-source-of-truth.md`

番号は作成順に連番。欠番は作らない。

---

## ADR のステータス

| ステータス              | 意味                              |
| ----------------------- | --------------------------------- |
| `Proposed`              | 提案中（まだ決定していない）      |
| `Accepted`              | 採用（現在有効）                  |
| `Deprecated`            | 廃止（後継 ADR に置き換えられた） |
| `Superseded by ADR-NNN` | ADR-NNN によって置き換えられた    |

---

## テンプレート

```markdown
# ADR-NNN: タイトル

**ステータス**: Accepted  
**日付**: YYYY-MM-DD  
**決定者**: ...

## 背景

なぜこの判断が必要だったか。どんな問題を解こうとしていたか。

## 判断

何を決めたか（1〜2 文で明確に）。

## 理由

なぜその判断をしたか。

## 却下した選択肢

検討したが採用しなかった選択肢と、その理由。

## 影響

この判断がもたらす制約・トレードオフ・注意点。
```
