## 次期開発計画（v0.9.x）— 2026-07-02 レビューによる再設計

**再設計の背景**: #027 完了時点でのレビューにより、「送信するまで」は CLI だけで完結する一方、
「送信した後」（返信 → 商談 → 成約/失注）は一切 CLI 化されておらず、シートを開いた手入力に
依存していることが判明した。ツールの核心（シートを開かなくていい）が営業として最も重要な
局面で崩れているため、**まず今の業務が CLI だけで回り切るようにする**ことを優先する。
ただし複数テンプレート管理は営業現場での利用頻度が高いため、レビュー時点の初期案から
前倒しし、leadScore 活用・AI パーソナライゼーションより先に着手する。

### Issue #032: 送信プレビュー強化（実装済み・参考）

**目的**: `send --preview` で「どのテンプレートを使うか」「誰に送るか」「件名」「本文」を送信前に分かりやすく確認できるようにする。複数社送信時でも一覧性を保ちつつ、1社だけ本文を見直したい・今回だけ送るのをやめたい、というニーズに応える。

**実装内容**

- `buildPreviewItems()`（対象/対象外の分類 + 本文生成）・`shouldSkip()`・`buildEmailContent()` は無改造で再利用
- 旧 `renderPreview()`/`confirmSend()` を廃止し、以下に置き換え:
  - `renderPreviewHeader()`: テンプレート表示名・内部名・送信対象/対象外/今回除外の件数を表示
  - `renderPreviewTable()`: 送信対象をコンパクトな1行テーブル（番号/企業ID/会社名/メール/件名を省略表示）で一覧表示。件数が多くても見やすいことを優先し、本文は表示しない
  - `renderTargetDetail()`: 番号指定した1社だけ件名・本文全文（省略なし）・HTML版有無を表示
  - `promptPreviewAction()` / `promptPreviewNumber()` / `promptDetailAction()`: 一覧画面「送信する/番号指定で確認/キャンセル」、詳細画面「一覧に戻る/**この企業を今回の送信対象から外す**」の対話プロンプト
- 本文確認画面から除外を選ぶと、その企業は**今回の送信対象からのみ**除外される（営業リストの送信可否等は一切変更しない）。一覧に戻ると送信対象件数が自動更新され、そのまま送信を続行できる
- `runPreview()` の戻り値を `boolean` から `{ confirmed, excludedCompanyIds }` に変更。`send.js` は `excludedCompanyIds` で送信ループの対象（`sendTargets`）を絞り込むだけ（Sheets への追加更新なし）
- `send.js` は既存の `templateName`/`scenarioName`（#031で解決済み）を `runPreview()` に渡すだけ（新規ロジックなし）

**変更ファイル**

- `src/cli/commands/send-preview.js`（表示層を全面更新。`buildPreviewItems`は無改造）
- `src/cli/commands/send-preview.test.js`
- `src/cli/commands/send.js`（`runPreview`呼び出しと`excludedCompanyIds`によるフィルタ追加）
- `src/cli/commands/send.test.js`（`--preview`時の除外ロジックのテスト追加）

**受入基準**: `sales-pilot send --preview` でテンプレート名・送信対象一覧・件名が確認でき、番号指定で1社の本文全文を確認できる。詳細画面から除外した企業は一覧の送信対象件数から減り、除外後もそのまま送信を続行できる。除外は営業リストのデータを変更しない

### Issue #036: leadScore 活用 + 送信後ダッシュボード自動更新

**目的**: Provider が算出するだけで未使用だった `leadScore` を実際の優先順位付けに使い、`send` 実行後もダッシュボードが最新状態を保つようにする（`check-replies` は既に更新している）

**実装内容**

- `find` の結果を `leadScore` 降順でソートして保存・表示
- `find --min-score <n>`: リードスコア閾値フィルタ
- `send` を `leadScore` 降順で送信（優先度の高い企業から連絡）
- `send` 完了後に `DashboardService.createOrUpdateDashboard()` を呼ぶ

**変更ファイル**

- `src/crawler/find.js`
- `src/cli/commands/find.js`
- `src/cli/commands/send.js`

**受入基準**: スコアの高い企業ほど `find` の表示・`send` の送信順で先に来る。`send` 実行後にダッシュボードが更新される
