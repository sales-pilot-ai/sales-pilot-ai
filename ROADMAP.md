## Issue #023〜#030 詳細仕様

---

### Issue #023: メールプレビュー・送信確認画面

**目的**: 送信前に誰に何を送るか確認して誤送信を防ぐ

**実装内容**

- `send --preview` / `-p` フラグを追加
- 企業ごとに「宛先・件名・本文冒頭3行」を表示
- `y/N` 確認プロンプトで送信実行または中断
- テンプレート変数が未定義の場合に警告を出力

**変更ファイル**

- `src/cli/commands/send.js`
- `src/gmail/template.js`

**受入基準**: `send --preview` で内容が表示され、`N` で中断・`y` で実際の送信に進む

---

### Issue #024: 送信ログ（JSONL 永続記録）

**目的**: 送信結果をファイルに永続記録し、過去の活動を追跡できるようにする

**実装内容**

- `logs/send-YYYY-MM-DD.jsonl` に 1 行 1 レコードで追記
- レコード形式: `{ companyId, companyName, email, status, timestamp, error }`
- 送信後サマリー表示: 「成功 N / 失敗 N / スキップ N」
- 失敗企業名を最後にまとめて列挙
- `logs/` を `.gitignore` に追加

**変更ファイル**

- `src/cli/commands/send.js`
- `src/utils/send-logger.js`（新規）
- `.gitignore`

**受入基準**: 送信後に `logs/send-YYYY-MM-DD.jsonl` が生成され、失敗時に警告が出る

---

### Issue #025: 返信検知（`check-replies`）

**目的**: 送信済みスレッドへの返信を自動検出してシートを更新する

**実装内容**

- 新規コマンド `check-replies`
- Gmail API `threads.list` で送信済みスレッドを検索
- 返信あり → シートの「返信有無」を自動更新
- OAuth スコープに `gmail.readonly` を追加
- 検出件数のサマリー表示

**変更ファイル**

- `src/cli/commands/check-replies.js`（新規）
- `src/gmail/reader.js`（新規）
- `src/cli/index.js`

**受入基準**: `check-replies` を実行すると返信のある企業の「返信有無」列が更新される

---

### Issue #026: 営業ダッシュボード（`report` コマンド）

**目的**: シートを開かずに現在の営業活動を CLI だけで把握できるようにする

**実装内容**

- 新規コマンド `report`
- ステータス別件数（未送信 / 送信済 / 返信あり / 成約）
- 今日・今週の送信数（送信ログ連携）
- 返信率（%）
- フォロー候補（送信から 7 日以上・返信なし）

**変更ファイル**

- `src/cli/commands/report.js`（新規）
- `src/cli/index.js`

---

### Issue #027: `find --review` 対話的確認モード

**目的**: 検索結果を 1 件ずつ確認してから保存できるようにする

**実装内容**

- `--review` / `-r` フラグ
- 企業ごとに「会社名 / URL / 電話 / スコア」を表示
- `s`=保存 / `x`=スキップ / `o`=保存+送信承認 / `q`=終了
- 選択結果をまとめて一括 append

**変更ファイル**

- `src/cli/commands/find.js`
- `src/crawler/find.js`

---

### Issue #028: AI パーソナライゼーション（Claude API 統合）

**目的**: 企業ごとに個別化した紹介文を生成して返信率を高める

**実装内容**

- `ANTHROPIC_API_KEY` 設定対応
- 企業情報 → Claude API → `introText` 生成
- API キーなし・エラー時はルールベース（`rules.js`）へフォールバック
- `--no-ai` フラグで強制ルールベース

**変更ファイル**

- `src/personalizer/ai.js`（新規）
- `src/personalizer/index.js`
- `src/cli/commands/diagnostics.js`

### Issue #030: スプレッドシート自動初期化 + `find` フィルタ強化
