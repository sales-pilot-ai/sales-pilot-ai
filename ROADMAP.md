### Issue #030: `find --review` 対話的承認モード

**目的**: 検索結果を自動保存せず、CLI上で1社ずつ確認してから営業リストへ保存できるようにする（find → シート編集での取捨選択 → send の往復をなくす）

**実装内容**

- `--review` / `-r` フラグ。指定時は `findCompanies()` 内での自動保存を止め（`skipSheets: true`）、レビュー後に承認分だけ保存する
- 企業ごとに「会社名 / メール / URL / 住所 / 電話 / 業種」を表示
- `inquirer` の `expand` タイプで `[y]保存 / [n]スキップ / [q]終了` を1キーで選択
- `q` で終了しても、それまでに `y` と判断した企業は保存対象として残す
- 判断ロジック（`reviewCompanies()`）は `promptDecision` を注入する純粋関数として実装し、実際のキー入力なしにテストできるようにする（CLIは表示のみ）
- 承認企業をまとめて1回で `appendCompanies()` に渡す（新規APIは作らない）
- サマリーに `appendCompanies()` の戻り値（`appended` / `merged`）をそのまま利用し、追加件数・重複更新件数・スキップ件数・残件数を表示する

**変更ファイル**

- `src/cli/commands/find-review.js`（新規、`reviewCompanies` / `formatCandidate` / `promptReviewDecision` / `printReviewSummary`）
- `src/cli/commands/find-review.test.js`（新規）
- `src/cli/commands/find.js`（`--review` 分岐の追加）
- `src/cli/commands/find.test.js`（review時のテスト追加）
- `src/cli/index.js`（`--review`/`-r` オプション登録）

**受入基準**: `sales-pilot find --review` で1件ずつ `y/n/q` を選択でき、`y` と判断した企業のみ営業リストへ保存され、完了時に追加件数・重複更新件数・スキップ件数・残件数が表示される

### Issue #032: leadScore 活用 + 送信後ダッシュボード自動更新

**目的**: Provider が算出するだけで未使用だった `leadScore` を実際の優先順位付けに使い、`send` 実行後もダッシュボードが最新状態を保つようにする（`check-replies` は既に更新している）

**実装内容**

- `find` の結果を `leadScore` 降順でソートして保存・表示
- `find --min-score <n>`: リードスコア閾値フィルタ
- `send` を `leadScore` 降順で送信（優先度の高い企業から連絡）
- `send` 完了後に `DashboardService.createOrUpdateDashboard()` を呼ぶ

**変更ファイル**

- `src/crawler/find.js`
- `src/cli/commands/find.js`
- `src/cli/commands/send.js`

**受入基準**: スコアの高い企業ほど `find` の表示・`send` の送信順で先に来る。`send` 実行後にダッシュボードが更新される
